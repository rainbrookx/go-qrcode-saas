# UI 风格提示词

## 整体风格

简洁、现代、功能导向的开源工具风格。参考 Linear / Vercel 的克制设计语言——大量留白、低饱和配色、清晰的层级关系。不追求视觉冲击，强调"打开即用、完成任务即走"的效率感。

---

## 配色体系

### 主色

| 角色 | 色值 | 用途 |
| ---- | ---- | ---- |
| Primary | `#1677FF`（Ant Design 默认蓝） | 主操作按钮、选中态、链接 |
| Primary Hover | `#4096FF` | 按钮悬停 |
| Primary Active | `#0958D9` | 按钮按下 |

### 中性色

| 角色 | 色值 | 用途 |
| ---- | ---- | ---- |
| 标题文字 | `#141414` | 页面标题、区块标题 |
| 正文文字 | `#434343` | 段落、说明文字 |
| 次要文字 | `#8C8C8C` | placeholder、辅助提示 |
| 边框 | `#D9D9D9` | 输入框边框、分割线 |
| 背景灰 | `#F5F5F5` | 页面底色、卡片区分区 |
| 白 | `#FFFFFF` | 卡片底色、输入框底色 |

### 语义色

| 角色 | 色值 | 用途 |
| ---- | ---- | ---- |
| Success | `#52C41A` | 创建成功、状态正常 |
| Warning | `#FAAD14` | 配额接近上限、即将过期 |
| Error | `#FF4D4F` | 校验失败、过期、错误 |

### 强调：短码 / 短链接

短码文本使用 Primary 色并加粗，使其在结果区域中第一眼可识别。

## 字体

| 层级 | 字号 | 字重 | 行高 | 场景 |
| ---- | ---- | ---- | ---- | ---- |
| H1 | 28px | 600 | 36px | 页面主标题（仅首页 Hero） |
| H2 | 20px | 600 | 28px | Tab 内容区标题 |
| H3 | 16px | 600 | 24px | 卡片内标题、弹窗标题 |
| Body | 14px | 400 | 22px | 正文、表单标签 |
| Caption | 12px | 400 | 20px | 辅助说明、时间戳、字数统计 |

字体栈：`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji'`（Ant Design 默认栈，不额外引入字体）。

代码/短码等宽场景使用：`'SFMono-Regular', Consolas, 'Liberation Mono', Menlo, Courier, monospace`。

## 布局

### 页面结构

```
┌─────────────────────────────────────────────┐
│ Header（Logo + 导航 + 登录/用户菜单）        │
├─────────────────────────────────────────────┤
│                                             │
│           主内容区 max-width: 960px          │
│           水平居中，垂直方向留白充足          │
│                                             │
├─────────────────────────────────────────────┤
│ Footer（开源信息、GitHub 链接）              │
└─────────────────────────────────────────────┘
```

### 内容区核心布局

```
┌──────────────────┬──────────────────┐
│   输入/配置区     │   预览/结果区     │
│   （左 50%）     │   （右 50%）      │
└──────────────────┴──────────────────┘
```

- 桌面端左右分栏，移动端上下堆叠（输入在上，预览在下）。
- 分栏断点：`768px`。

### Tab 栏

- 使用 Ant Design `Tabs` 组件，`type="line"`，居中对齐。
- Tab 顺序：文本 → 网址静态码 → 网址跳转码 → 文章 → 表单 → 解码。
- 每个 Tab 对应独立路由（如 `/text`、`/url`、`/urldyn` 等），URL 与 Tab 同步。

## 组件风格约定

### 按钮

- 主按钮：`<Button type="primary">`，仅页面最核心操作使用（如"生成二维码"、"提交"）。
- 次按钮：`<Button>` 默认样式，用于"复制链接"、"下载"等辅助操作。
- 文字按钮：`<Button type="link">`，用于"编辑"、"删除"等行内操作。
- 危险操作：`<Button danger>`，用于"删除"确认。
- 按钮最小高度 `32px`，主操作按钮宽度撑满或固定 `120px`。

### 输入框

- 使用 Ant Design `Input` / `Input.TextArea` / `Select` / `DatePicker`。
- 统一 `size="middle"`。
- 网址输入框前缀带 `http(s)://` placeholder 提示（仅网址静态码 Tab）。
- 校验错误信息紧贴输入框下方，红色文字 `12px`。

### 卡片

- 预览区使用 `Card`，`bordered`，圆角 `radius-md`。
- 卡片阴影：`0 1px 2px 0 rgba(0, 0, 0, 0.03), 0 1px 6px -1px rgba(0, 0, 0, 0.02), 0 2px 4px 0 rgba(0, 0, 0, 0.02)`（Ant Design 默认）。
- 不使用粗重阴影或彩色边框。

### 二维码预览

- 二维码居中展示在预览卡片内。
- 下方紧跟操作按钮行："复制链接" / "下载 PNG"，水平排列，间距 `8px`。
- 二维码图片固定展示尺寸 `200×200px`，下载输出原始分辨率 PNG。

### 表格

- 我的活码列表使用 Ant Design `Table`，`size="middle"`，`bordered`。
- 列：类型 Tag、短链接、创建时间、过期时间、状态、操作。
- 空状态使用 `Table` 自带 `locale.emptyText`。

### Tag

- 活码类型用彩色 Tag 区分：
  - 网址跳转码：蓝色 `blue`
  - 文章：绿色 `green`
  - 表单：橙色 `orange`
- 状态 Tag：
  - 活跃：绿色
  - 已过期：红色
  - 即将过期（≤3天）：黄色

### 弹窗 / 抽屉

- 编辑操作使用 `Modal`，宽度 `520px`。
- 表单构建器使用 `Drawer`，`width={640}`，从右侧滑出。
- 确认删除使用 `Modal.confirm`。

### 消息提示

- 操作成功：`message.success`
- 操作失败：`message.error`
- 配额警告：`notification.warning`（需用户注意但不阻断）

---

## 登录 / 注册页面

- 居中卡片式布局，卡片宽度 `400px`。
- 包含：Logo、标题、邮箱输入、密码输入、主按钮。
- 注册与登录通过 Tab 切换（Ant Design `Tabs`，`size="small"`）。
- 找回密码为独立页面或当前页内切换表单。

## 各 Tab 布局细则

### 文本 / 网址静态码（无需登录）

```
┌──────────────────┬──────────────────┐
│ TextArea / Input │   QR 预览卡片    │
│ + 字数统计       │   + 下载/复制    │
│ + 生成按钮       │                  │
└──────────────────┴──────────────────┘
```

- 文本 Tab：`TextArea`，显示实时字数 `x / 最多字数`。
- 网址静态码 Tab：`Input`，prefix 显示锁图标，placeholder `http(s)://`。

### 网址跳转码（需登录）

```
┌──────────────────┬──────────────────┐
│ 目标 URL Input   │   QR 预览卡片    │
│ 有效期选择       │   短链接（可复制）│
│ + 生成按钮       │   + 下载/复制    │
└──────────────────┴──────────────────┘
```

- 有效期默认 30 天，可选自定义，最长 60 天，使用 `DatePicker` 或预设选项。
- 生成后展示短链接，点击可复制。
- 未登录时，输入区上方显示提示条："登录后可创建活码并修改目标链接"，附带登录按钮。

### 文章（需登录）

```
┌────────────────────────────────────────┐
│ TipTap 富文本编辑器（全宽）             │
│ 工具栏：加粗/斜体/标题/列表/图片/视频/  │
│        音频/附件/链接                   │
├────────────────────────────────────────┤
│ 底部栏：字数统计 | 有效期 | 生成按钮    │
├────────────────────────────────────────┤
│ 预览区：QR 码 + 短链接 + 下载/复制      │
└────────────────────────────────────────┘
```

- 编辑器全宽展示，最小高度 `400px`。
- 底部栏固定在编辑区下方，显示字数 `x / 50,000`。
- 附件上传按钮在工具栏内，上传后以附件列表形式展示在编辑区下方。
- 上传进度使用 Ant Design `Upload` 组件自带进度条。
- 文件大小超限时，即时红色提示，阻止上传。

### 表单（需登录）

```
┌──────────────────┬──────────────────┐
│ 表单构建器        │  实时预览         │
│ 拖拽字段列表      │  表单渲染结果     │
│ 字段属性配置      │                  │
│ + 有效期/提交上限 │                  │
│ + 生成按钮        │  QR + 短链接     │
└──────────────────┴──────────────────┘
```

- 左侧：字段类型面板（可拖拽） + 已添加字段列表（可拖拽排序） + 选中字段属性编辑。
- 右侧：实时渲染表单预览 + 二维码结果。
- 支持字段类型：单行文本、多行文本、单选、多选、下拉、日期、文件上传、评分。

### 解码（无需登录）

```
┌────────────────────────────────────────┐
│         上传区域（居中，拖拽或点击）      │
│         最大 5 MB                       │
├────────────────────────────────────────┤
│ 解码结果：文本内容（可复制）             │
└────────────────────────────────────────┘
```

- 上传区域使用 `Upload.Dragger`，居中，宽度 `480px`。
- 解码结果展示在下方，纯文本内容，带复制按钮。
- 非二维码图片提示错误。

## 空状态与缺省

| 场景 | 展示 |
| ---- | ---- |
| 我的活码列表为空 | 居中插图 + "还没有创建活码，去创建第一个" + 跳转按钮 |
| 搜索无结果 | "未找到匹配的活码" |
| 表单尚无提交 | "暂无提交数据" |

---

## 加载与过渡

- 按钮点击后立即进入 `loading` 态（Ant Design Button `loading` 属性），防止重复提交。
- 页面级加载使用 Ant Design `Spin`，居中展示。
- Tab 切换无过渡动画，即时切换（保持响应感）。
- 二维码生成使用 `Spin` 覆盖预览区，生成后淡入展示。

---

## 响应式策略

| 断点 | 行为 |
| ---- | ---- |
| ≥ 768px | 左右分栏布局，Header 水平导航 |
| < 768px | 上下堆叠，Header 折叠为汉堡菜单，Tab 改为横向滚动 |

- 所有输入控件和按钮在移动端撑满宽度。
- 二维码预览尺寸在移动端缩至 `160×160px`。
- 文章编辑器在移动端保持全宽，工具栏可横向滚动。

---

## 动效

- 不添加装饰性动效。
- 仅保留功能性反馈：按钮 hover/active 状态过渡（`transition: all 0.2s`）、Modal 进出动画（Ant Design 默认）、Toast 出现/消失。

---

## 无障碍

- 所有可交互元素可键盘聚焦，焦点环使用 Primary 色 `2px solid` 轮廓。
- 图片提供 `alt` 文本（二维码 alt 为 "二维码"）。
- 表单输入关联 `label`。
- 颜色对比度满足 WCAG AA（4.5:1 正文，3:1 大字）。

---

## 暗色模式

- MVP 阶段**不支持暗色模式**。
- 配色体系仅定义 Light 模式。
